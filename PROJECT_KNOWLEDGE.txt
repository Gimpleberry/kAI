4. PLUGIN REGISTRY (BOOT ORDER)
═══════════════════════════════════════════════════════

  Defined in plugins.py. Loaded in this order at boot, reversed at shutdown.

  1. StoragePlugin       Database schema + connection
  2. TaxonomyPlugin      Loads + caches the validated taxonomy
  3. DesignPlugin        CBC + MaxDiff generation lifecycle
  4. EstimationPlugin    MNL + ensemble lifecycle
  5. ProfilePlugin       Profile generation + export lifecycle
  6. ElicitationPlugin   FastAPI server (last — depends on all others)

  Each plugin implements start() and stop() (Tenet 1). The contract is
  validated by:
    - python main.py --check
    - tests/unit/test_plugins_registry.py
    - bash scripts/validate.sh (item #4 + #6)

6. ARCHITECTURE RULES (CURRENT)
═══════════════════════════════════════════════════════

  TENET 1 (SCALABLE)
    - All cross-cutting symbols live in shared.py
    - Each feature is a plugin module with start()/stop()
    - Adding a feature = create module + one line in plugins.py
    - Layered architecture: STORAGE → REFRESH → API → UI

  TENET 2 (EFFICIENT)
    - External calls cached with TTL + audit log (when added)
    - Delta detection before writing snapshots
    - Heavy startup work deferred to post-boot jobs
    - Per-item exception isolation: batch_with_isolation() in shared
      for any batch where one bad item must not abort the run.

  TENET 3 (SAFE)
    - All secrets via load_local_config() — fail loud if missing
    - API binds 127.0.0.1 only
    - .env never committed; pre-commit hook + CI enforce
    - Action safety enforced per ADR-009: automation prepares,
      human commits. ActionSafetyError available in shared.py.
    - Secret-masking via mask_secret() in shared (ADR-010).

  TENET 4 (ORGANIZED)
    - Pinned deps (Python 3.12.x, all libs at exact versions)
    - CHANGELOG entry for every meaningful change
    - DECISIONS.md ADR for every architectural choice
    - Phase-ordered BACKLOG.md
    - Versioning scheme + tag taxonomy per ADR-011.
    - 11-step rollout via scripts/rollout.sh.

  TENET 5 (PRIORITY QC)
    - 10-item validation checklist (scripts/validate.sh) before every push
    - Tests in tests/unit run against sandboxes, never production data
    - Every bug fix gets a regression test
    - Structured logging with rotation per ADR-010
      (setup_rotating_logger in shared).

  TENET 6 (STORAGE)
    - SQLite for queryable data, JSON for whole-file blobs
    - Universal columns at top level (id, created_at, updated_at)
    - Snapshots for time-series KPIs, with delta detection
    - Schema initialization is idempotent
    - WAL mode for concurrent reads

9. KNOWN ISSUES & TECHNICAL DEBT REGISTER
═══════════════════════════════════════════════════════

  Per ARCHITECTURE_TENETS doc-distribution: this register lists
  acknowledged-but-unfixed bugs and known limitations, each with its
  current workaround. Distinct from BACKLOG.md (which is forward-looking
  new work). Items move FROM here INTO BACKLOG when scheduled for a fix.

  When adding an entry: state the SYMPTOM, the WORKAROUND if any, and
  the CONDITIONS under which it would matter enough to fix.


  CURRENTLY EMPTY — no acknowledged debt as of v0.2.1.

  This is a deliberate state, not an oversight. Document any debt the
  moment it's introduced. Examples of what would go here when it arises:
    - "Plugin Y currently uses module-level state instead of a class
       attribute; refactor pending Phase N. Workaround: don't run two
       Plugin Y instances simultaneously (we don't anyway)."
    - "MNL bootstrap is single-threaded; takes ~30s for 500 iterations.
       Acceptable now; revisit if iteration count grows past 2000."
    - "FastAPI app does not yet bind on Python 3.13 due to <library>
       incompatibility. Workaround: stay on 3.12 (already required by
       ADR-007)."

  ITEM 3 (functional): Phase 1.1 generator falls below D-efficiency gate
    Symptom:    diagnose_cbc_design() reports D-eff ~0.38 on production
                config (20 tasks x 4 alts x 8 attrs x 3 levels). Gate
                threshold is 0.85. passes_gates returns False.
    Cause:      Phase 1.1 implements level-balanced independent shuffles
                only. Calibration shows this is statistically
                indistinguishable from pure random sampling. The 0.85
                gate is calibrated against full Sawtooth balanced overlap
                which uses swap-based D-eff optimization.
    Workaround: Sentinel test in tests/unit/test_design_diagnostics.py
                (test_production_design_intentionally_fails_d_eff_gate)
                asserts the failure with a docstring explaining the path
                forward. Estimation still works at D-eff 0.38 (wider CIs,
                not wrong answers), so the intermediate state is survivable.
    Fix cost:   BACKLOG item 1.1.5 - swap-based optimization on top of
                level-balanced sampling. Should land before Phase 1.4
                (MNL estimator).
    Priority:   High - should ship before MNL estimation builds on top
                of the current design.
